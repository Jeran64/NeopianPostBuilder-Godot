Run the NeopianPostBuilder.exe or NeopianPostBuilder.sh to execute the program.

1. To add entries, select a type. (Different types will require different information to be considered complete.)
2. Fill in all of the data as relevant.
3. Hit "Save Entry" to save the entry. 
    * Incomplete entries will save to the "Incomplete" category and can be completed later.
    * Complete entries will save to thier indicated categories.
4. To edit an entry, click on the entry in the category. 
    *Hit Save to save any changes.
    *Hit Cancel to cancel any changes, or escape to make a new entry.
    *Hit Delete to delete the entry from the issue. (There is no undo!)
5. Click "Save Issue" to save the progress of building the current issue. Pick a file name (ex:"Issue15") to save it as a .txt file.
    * Hit "Load Issue" to load a previously saved issue, and resume where you left off.
6. Fill in the Issue#
7. Pick a directory to save the issue's folder of contents. This folder needs to also contain the associated Archives.html
8. If per step 7, your chosen directory does not have an Archives.html, enable the "First Time Setup" Option. An Archives.html will be copied into the destination folder.
9. Once 3 or more entries have been added, click "Pcik Frontpage" to select 3 random front page entries. 
10. Decide on the Quote of the Issue, and enter it into the associated text box. Include attribution in the Author section.
11. Verify all of your information is accurate.
12. Hit Publish!
13. Drag and drop the Issue Folder, and the Archives.html into the main folder of the webhost folder. The folder will be added, and you want to overwrite the old Archives.html with the new Archives.html.
14. Update the home index.html page. 
    1. Update the latest issue link
    2. Update the new deadlines.
